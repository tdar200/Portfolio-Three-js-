Inventory Management System

I built an advanced Inventory Management System designed to streamline and optimize inventory management for businesses with integrated POS systems. The system features multi-tier authentication, granting role-based access to Admins and Customers, ensuring secure operations. Real-time updates are facilitated through API and webhook integration with the POS system, allowing automatic subtraction of ingredients based on product sales data. Admins can add purchased items, view detailed inventory history, and manage fixed costs like salaries, providing accurate overall business expense calculations. The system calculates average product costs and profit margins using historical purchasing prices and current sales data. It generates daily cash sheets with comprehensive sales, profit, and cost data. The e-commerce functionality includes product listings, a cart system, and a checkout process integrated with Stripe for payment processing. The tech stack comprises React, Vanilla JavaScript, Redux, Bootstrap, MUI, Context API, Express, MongoDB, Webhooks, and REST API. This robust solution ensures real-time inventory management, detailed financial insights, and efficient e-commerce operations.

Finding Waldo

I developed a game using a variety of modern web technologies to create an engaging and interactive experience. The frontend is built with HTML5, CSS3, and JavaScript, providing a responsive and visually appealing interface. The backend is powered by Node and Express, with MongoDB as the database. For database operations, I used both Mongoose for schema-based modeling and MongoClient for specific interactions, such as fetching images. The game employs await/async for efficient promise handling, ensuring smooth and responsive gameplay. The game's logic includes offset x and y positions upon clicks and predefined values to determine if a character has been clicked. When the user starts the game, a timer begins, incrementing every second using the setInterval method, and stops once all characters are found. The number of remaining characters is displayed in the top NavBar for easy tracking. The game images are fetched from the backend using MongoClient, showcasing a dynamic content delivery approach. This project combines real-time updates and persistent data storage, creating a seamless and immersive user experience.

https://finding-waldo.herokuapp.com

Tic-Tac-Toe

I developed a Tic-Tac-Toe game using HTML5, CSS3, and JavaScript, leveraging functional components to create a dynamic and interactive experience. The game features a 2-player option and maintains a score count for each player. DOM manipulation is achieved through the use of query selectors and Event Listeners, allowing for real-time interaction and updates. The game logic checks for winning conditions using a pre-defined array of possible combinations. Higher-order functions like some and every are employed to efficiently loop through and verify if any of the provided combinations match the current state of the board. When a player wins or the game ends in a draw, a message is displayed with an option to restart the game. Upon restarting, the board is reset, and a point is added to the winning player's score, ensuring a seamless and engaging gameplay experience. This project demonstrates the effective use of vanilla JavaScript for creating a functional and enjoyable Tic-Tac-Toe game.

https://tdar200.github.io/Tic-Tac-Toe
